# 相关依赖的安装指南（Windows）

本指南适用于 `interview_voice_project_refactor`。

## 1. 创建 Conda 环境（Python 3.11）

```powershell
conda create -n interview_voice_project_refactor_py3_11 python=3.11
conda activate interview_voice_project_refactor_py3_11
```

## 2. 安装项目依赖（requirements.txt）

在重构项目目录执行：

```powershell
pip install -r requirements.txt
```

## 3. 手动安装 FFmpeg

### 3.1 下载 FFmpeg

1. 访问官方构建站：
   - https://www.gyan.dev/ffmpeg/builds/
2. 下载：`ffmpeg-release-essentials.zip`
3. 解压到固定目录，例如：
   - `C:\ffmpeg`

解压后应包含：
```
C:\ffmpeg\bin\ffmpeg.exe
C:\ffmpeg\bin\ffprobe.exe
```

### 3.2 配置环境变量（PATH）

**方式 A：图形界面**

1. 打开 **系统属性 → 高级 → 环境变量**
2. 在“系统变量”里找到 `Path`，点击编辑
3. 新增一条：
   - `C:\ffmpeg\bin`
4. 保存并关闭所有窗口

**方式 B：命令行（管理员 PowerShell）**

```powershell
setx /M PATH "$env:PATH;C:\ffmpeg\bin"
```

> 注意：执行后需重启终端或重启系统

### 3.3 验证安装

打开新的终端执行：

```powershell
ffmpeg -version
```

如果输出版本信息，则安装成功。

## 4. 常见问题

- **提示找不到 `ffmpeg` 命令**
  - 确认 `C:\ffmpeg\bin` 已加入 PATH
  - 关闭并重新打开终端

- **多版本冲突**
  - 确保 PATH 中只有一个 ffmpeg 版本路径
  - 可以通过 `where ffmpeg` 查看实际生效路径

```powershell
where ffmpeg
```

## 5. 项目提示

`interview_voice_project_refactor` 在处理音频（分割、转换）时依赖 FFmpeg。
确保在启动 Worker 前完成安装。
