# Streamlit UI 重构服务（隔离版）

此目录用于放置 **重构后的 Streamlit UI**，不影响原项目。

## 结构

```
interview_voice_project_refactor/
├── apps/
│   └── ui/

│   ├── main.py
│   └── pages/
│       ├── __init__.py

│       ├── page_main.py
│       └── page_test.py
├── core/

```

## 启动方式

```bash
streamlit run app/main.py
```

## 配置

- `API_BASE_URL`：后端 API 基础地址（默认 `http://127.0.0.1:8001`）

## 主要体验

- 上传成功后直接进入记录详情页，可查看处理状态。
- 上传页会先校验必填项、文件格式、文件大小和后端服务状态。
- 上传失败时会展示错误代码、错误类型、trace_id 和可操作建议。
- 上传页保留最近提交任务入口，便于返回刚提交的记录详情。
- 首页支持按姓名、公司、学科、记录 ID 搜索，按状态/疑似卡住筛选，并按更新时间、最近进度、创建时间、面试时间排序。
- 首页和详情页会展示 Worker 任务追踪 ID，便于对照后台日志定位失败原因。
- 未完成记录可在详情页查看标准化阶段进度条，并默认自动刷新状态。
- 详情页会展示已处理时长、当前阶段停留时长和最近进度更新时间，处理中任务长时间无更新时会提示可能卡住。
- 详情页会展示 ASR 分片 DB 缓存、兜底文件缓存和继续处理建议，辅助判断是否需要重新上传。
- 失败记录会优先展示结构化失败原因、错误代码、错误类型、重试次数、失败时间和处理建议。
- 失败记录可点击“继续处理”，不需要重新上传录音。
