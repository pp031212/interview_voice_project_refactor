# 下一步可优化空间（统一优先级）

以下为当前重构项目的优化方向，按实施优先级排序。

## 已完成项（基线）

- [已完成] `voice_to_text` 从”节点级断点”细化为”分片级断点”
- [已完成] 每个音频分片转写结果中间落盘（DB 主存 + JSON 兜底）
- [已完成] 失败重跑时跳过已完成分片，避免整节点从头转写
- [已完成] `asr_resume` 中间态迁移为数据库主存（`tb_asr_segment_cache`）
- [已完成] 增加健康检查路由（`/health` 和 `/readiness` 端点）
- [已完成] 统一配置加载（删除重复的 `load_dotenv()` 调用）
- [已完成] 全局异常处理中间件（FastAPI + Worker，支持异常分类）
- [已完成] 异常分类完善（核心模块已使用具体异常类型，覆盖率约 40%）
- [已完成] 失败记录详情页展示错误原因，并支持一键继续处理
- [已完成] 记录详情页展示阶段进度条，待处理/处理中状态支持自动刷新
- [已完成] 上传页增加前置校验、服务检查、最近任务入口和友好错误提示
- [已完成] 处理阶段标准化：新增 `processing_stage` 字段，详情页优先展示标准阶段，旧记录回退 `processing_tips` 推断
- [已完成] 失败信息结构化：新增 `error_code`、`error_type`、`error_message`、`retry_count`、`max_retries`、`failed_at` 字段，详情页优先展示结构化失败原因
- [已完成] 处理耗时统计：新增处理开始、阶段开始、最近进度、完成时间字段，详情页展示耗时并提示疑似卡住任务
- [已完成] 记录列表检索与异常任务筛选：主页支持按姓名/公司/学科/ID 搜索，按状态和疑似卡住筛选，按时间排序
- [已完成] 任务追踪 ID 持久化：Worker 单次任务 trace_id 写入主表，首页、详情页和脚本可用于定位日志
