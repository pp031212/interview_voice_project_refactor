# 下一步可优化空间（统一优先级）

以下为当前重构项目的优化方向，按实施优先级排序。

## 已完成项（基线）

- [已完成] `voice_to_text` 从”节点级断点”细化为”分片级断点”
- [已完成] 每个音频分片转写结果中间落盘（DB 主存 + JSON 兜底）
- [已完成] 失败重跑时跳过已完成分片，避免整节点从头转写
- [已完成] `asr_resume` 中间态迁移为数据库主存（`tb_asr_segment_cache`）
- [已完成] 增加健康检查路由（`/health` 和 `/readiness` 端点）
- [已完成] 统一配置加载（删除重复的 `load_dotenv()` 调用）
- [已完成] 全局异常处理中间件（FastAPI + Worker，支持异常分类）
- [已完成] 异常分类完善（核心模块已使用具体异常类型，覆盖率约 40%）

## P0（短期，稳定性优先）

- 统一日志与追踪（请求/任务级别 trace_id）
- 任务状态机标准化（processing_status + error_type）
- Worker 重试策略（可配置次数、退避）
- [已完成脚本层] 为分片级断点增加 TTL 清理策略与状态查询（`scripts/manage_asr_resume_cache.py`，UI 可视化另行推进）

## P1（中期，质量与效果提升）

- 建立本地评分 Rubric（维度、权重、评分锚点）替代纯模型主观打分
- 将评分拆分为“LLM 抽取证据 + 本地规则聚合总分”
- 增加跨模型评分校准层（GPT / DeepSeek 等输出映射到统一分布）
- 引入评分稳定性评估（同记录多次评估方差监控）
- 增加领域术语词典/热词机制（公司名、岗位名、专业名词）
- 增加 ASR 二次纠错（基于术语词典与上下文的后处理）
- 增加音频质量检测与降噪预处理（口音/噪声场景）
- 增加“低置信度片段”标记与人工复核入口
- 评估并接入科大讯飞 ASR 能力（API/流式）作为可选引擎
- 设计可插拔 ASR Provider 抽象层，支持 FunASR/讯飞等多引擎切换
- 增加双引擎策略：主引擎转写 + 低置信度片段二次复核（兼顾质量与成本）
- 进一步拆分 API：加入专用 `schemas` 的响应模型字段约束（原搁置项）

- 前端进度条（显示当前节点）

- 数据库记录查看/检索条件
- API 集成测试（pytest + httpx）
- pipeline 单元测试（节点级）
- DB 层 mock 测试
- 完善测试用例（pytest + API 测试，原搁置项）
